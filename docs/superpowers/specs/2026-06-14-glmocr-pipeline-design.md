# GLM-OCR Smart Pipeline 设计规格说明书

本设计书旨在为 GLM-OCR 离线部署管道构建一套工业级、高鲁棒性且高推理速度的文档智能解析与 OCR 工作流。通过将经典的传统图像算法与前沿的多模态大语言模型（GLM-OCR）相结合，实现对复杂排版（如双栏学术论文、公式、表格等）的精确解析。

---

## 1. 背景与目标

虽然通用多模态大模型（VLM）本身具备直接读取倾斜和复杂版面图片的能力，但在“版面分析 + 物理裁剪 + 局部 OCR”的两阶段（Two-stage）工程工作流中，前置的几何处理与后置的拼接重排依然是决定识别精度和速度的关键。
本工作流旨在解决以下痛点：
1. **边缘字符截断**：版面模型输出的紧贴型矩形框物理裁剪时容易切断字符笔画，导致 VLM 识别错误。
2. **邻栏文字干扰（串扰）**：VLM 对裁剪图边缘多出的邻近文字极其敏感，容易产生复读和混淆幻觉。
3. **逻辑流错乱**：双栏文档在没有几何排序时拼接会导致文字左右交叉。
4. **串行推理缓慢**：逐个碎片发送 API 导致 GPU 闲置与严重的时延。

---

## 2. 模块划分与接口设计

整个工作流共划分为六个核心处理模块，各模块之间高内聚、低耦合，通过标准的数据结构进行交互。

### 2.1 页面图像预处理与倾斜纠偏模块 (De-skewing)
*   **输入**：由 `fitz` (PyMuPDF) 从 PDF 渲染出来的 $300\text{ DPI}$ 原始页面图像 `Image`。
*   **处理逻辑**：
    1. 使用 OpenCV 霍夫线变换（Hough Line Transform）或投影直方图最大方差法，检测图像中文字行的全局倾斜角 $\theta$。
    2. 若 $\theta > 0.5^\circ$，利用仿射变换矩阵（`cv2.warpAffine`）将图像整体旋转 $-\theta$ 进行拉平摆正，旋转边缘的空白区域用纯白色（RGB 255, 255, 255）填充。
*   **输出**：摆正后的页面图像 `Image`。

### 2.2 版面检测模块 (Layout Detection)
*   **输入**：摆正后的页面图像 `Image`。
*   **处理逻辑**：
    1. 调用本地 PyTorch 版 `PP-DocLayoutV3` (safetensors) 模型进行目标检测。
    2. 过滤置信度低于 $0.4$ 的噪声框。
*   **输出**：Bounding Boxes 列表，结构为：
    ```python
    [
        {"coords": [xmin, ymin, xmax, ymax], "label": "text/table/formula/figure"},
        ...
    ]
    ```

### 2.3 阅读顺序重排模块 (XY-Cut Reader Order)
*   **输入**：无序 of Bounding Boxes 列表。
*   **处理逻辑**：
    1. 采用经典的 **XY-Cut（递归投影切分法）**，沿水平（X）和垂直（Y）方向上的投影空白区域进行递归剖分。
    2. 建立排版剖分树，对其进行深度优先遍历（DFS）。
*   **输出**：按照符合人类逻辑阅读顺序（如双栏排版下先左栏后右栏）重排后的 Bounding Boxes 序列。

### 2.4 智能裁剪与邻近去噪模块 (Masked Padding)
*   **输入**：摆正后的页面图像 `Image`，重排后的有序 Bounding Boxes 序列。
*   **处理逻辑**：
    1. **插图处理 (`figure`)**：若类型为 `figure`，直接物理裁剪该框，并保存至 `output_dir/images/page_X_fig_Y.png`。不进行大模型 OCR，在最终拼装流中直接保留 `![figure](images/page_X_fig_Y.png)`。
    2. **文本类处理 (`text`/`table`/`formula`)**：
       * **动态外扩（Padding）**：计算框的尺寸，横向外扩宽度的 $2\%$，纵向外扩高度的 $3\%$（且最少外扩 $\pm 6\text{px}$，不超过图像边界）。
       * **冲突去噪（Masking）**：检测该外扩后的 padded 框是否与相邻的其他任何 Bounding Box 重叠。若重叠，在裁剪出的子图上，将属于其他 Box 坐标范围的图像区域直接填充为纯白色（RGB 255, 255, 255）。
       * **超分辨率插值（Resizing）**：若裁剪出的子图高度小于 $64\text{px}$，使用 Bicubic 插值等比例放大至高度 $128\text{px}$。
*   **输出**：生成的干净局部子图像文件列表，以及不需 OCR 的图片 Markdown 标记。

### 2.5 异步并发 OCR 推理模块 (Async vLLM Engine)
*   **输入**：子图像文件列表。
*   **处理逻辑**：
    1. 利用 `asyncio` 和 `aiohttp`，向本地 vLLM 接口（端口 8700）发起异步 HTTP POST 请求。
    2. 限制最大并发数通道为 **`4`**，以匹配 4080 Laptop (12GB) 的硬件负荷，防止 OOM。
    3. 针对不同类型子图，在 Prompt 中传入特定的视觉指引词：
       * 普通文本段落/标题：`"Text Recognition:"`
       * 表格：`"Table Recognition:"`
       * 公式：`"Formula Recognition:"`
    4. 设定 `temperature = 0.0` 确保输出一致性，并对不同类型限定 `max_tokens` 抑制大模型因幻觉产生无意义的复读。
*   **输出**：识别完成的文本段落列表。

### 2.6 文本拼接与后处理模块 (Postprocessing)
*   **输入**：OCR 识别出的文本段落以及无需 OCR 的图片/表格标记。
*   **处理逻辑**：
    1. 按照 XY-Cut 重排后的有序索引，将文本拼接成初步的 Markdown。
    2. 调用已有的 `smart_reflow_markdown` 进行单页内的行合并与特殊标签保护。
    3. 如果是多页文档，调用 `stitch_pages` 进行跨页无缝缝合。
    4. 调用已有的 `convert_file_with_pandoc` 使用模板导出标准的 DOCX。
*   **输出**：排版优雅、逻辑顺序正确的 Markdown 和 Word 文件。

## 3. TDD（测试驱动开发）验证计划

为了保障系统的稳定性，我们在编写主流程前将引入自动化单元测试：

| 测试文件 | 目标模块 | 测试用例与断言内容 |
| :--- | :--- | :--- |
| `tests/test_deskew.py` | 倾斜纠偏 | 输入倾斜 $5^\circ$ 的标准图，断言检测出的倾斜角为 $5.0^\circ \pm 0.5^\circ$，且摆正后倾斜误差 $< 0.5^\circ$。 |
| `tests/test_xycut.py` | XY-Cut 排序 | 输入模拟双栏的乱序坐标列表，断言输出顺序必须为先左栏后右栏的阅读顺序。 |
| `tests/test_masking.py` | 去噪遮罩 | 输入两个有重叠的相邻 boxes，断言裁剪子图中重叠区域像素的平均值被成功刷白（RGB 255）。 |
| `tests/test_async_ocr.py` | 并发推理 | 模拟 4 个子图并发，断言在设定超时（15秒）内全部请求成功（200 OK）且非空。 |

## 4. 容错与优雅降级机制

1. **API 超时处理**：单个异步请求超时设为 15 秒，提供指数退避（Exponential Backoff）重试机制，最大重试 3 次。
2. **块级失败隔离**：如果重试 3 次后该子图仍无法解析，整个 PDF 转换管道**不断开**。在最终 Markdown 对应的位置插入占位符：`[OCR识别失败：该区域图像已保存至 crop_failed_page_X_block_Y.png]`，并把该裁剪子图输出到指定目录，日志中记录错误，以便用户进行人工核对。
